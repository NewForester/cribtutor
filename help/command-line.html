For a description of the command line options in simple English just hit return

In principle, `-d <dir>`; is the only option needed to run a quiz.

The cribtutor program generates a quiz based on the contents of a list of crib-sheets.

By default, the crib-sheets are listed in a file named "cribsheets.txt". Use `-f <file>` to specify an alternative.

Typically, "cribsheets.txt" and the the crib-sheets it lists will be in the same directory. Use `-d <dir>` to specify this directory. Typically, the directory names the quiz.

If `-d <dir>` does not specify an absolute pathname then `dir` is relative to the directory that contains the program (which may not be `pwd`).

By default, a quiz starts with the first crib-sheet listed in "cribsheets.txt" but the quiz can be started with any crib-sheet using the `-s` option to specify the crib-sheet by name.

Only the first few characters of the crib-sheet name are needed. For larger quizzes split into 'chapters' and 'sections', the crib-sheet name typically begins with the 'chapter'/'section' number so: 

    -s 2        // begin with chapter 2
    -s 23       // begin with section 3 of chapter 2

Statements in crib-sheets (usually) contain terms the program blanks out when posing a question. By default, the program will blank out two terms in each statement. Use `-c <n>` to blank out a different number of terms.

Note: To read crib-sheets without questions and answers, use `-c 0`

Use `-h` to enter this tutorial but then you already knew that.

Use `-t` to use test mode. Test mode runs short quizzes found in the test directory.

Use `-p` to just parse and print a crib-sheet and exit. No questions and answers, no prompts. Used to test/debug crib-sheets.

Use `-p -r` to just parse and print the parse tree and exit. Used to debug the crib-sheet parser.
